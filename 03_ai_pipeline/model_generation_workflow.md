# Model Generation Workflow

## 1. 생성 전략

생성형 3D 모델은 무조건 처음부터 만들지 않는다. 실제 프로젝트에서는 “정확성, 구매 가능성, 속도”가 중요하므로 다음 우선순위를 따른다.

```text
1. 실제 판매 제품 검색
2. 승인된 에셋 라이브러리 검색
3. 유사 에셋 변형
4. 새 3D 모델 생성
5. 디자이너 수동 보정
```

## 2. Candidate Generation

### 입력 예시
- 공간: 38㎡ 카페, 창가 6m, 입구 폭 1.4m.
- 사용자 의도: “북유럽풍, 밝은 원목, 2인석 3개, 통로 넓게.”
- 스케치: 창가 쪽 3개 사각형, 입구에서 카운터까지 동선 화살표.

### 생성 후보
1. 실제 브랜드 테이블+의자 조합 3개.
2. 일반 에셋 라이브러리 조합 2개.
3. 새로 생성한 콘셉트 의자 2개.
4. 레이아웃 변형 5개.

## 3. Asset Normalization

생성/수집 에셋은 바로 사용할 수 없다. 아래 정규화가 필요하다.

| 처리 | 목적 |
|---|---|
| Scale calibration | 실제 치수와 일치 |
| Pivot correction | 이동/회전 조작 자연화 |
| Collider generation | 충돌 검사 |
| Bounding box extraction | 배치/동선 검증 |
| PBR material tagging | 렌더 품질 및 재질 견적 |
| LOD generation | XR 성능 최적화 |
| Metadata enrichment | 가격, 브랜드, 소재, 납기 연결 |

## 4. Quality Gate

생성 결과는 다음 기준을 통과해야 XR 장면에 표시된다.

- 삼각형 수가 디바이스 성능 기준 이하.
- 실제 치수 범위가 합리적.
- 바닥 접지면이 안정적.
- 뒤집힌 노멀/깨진 메쉬가 없음.
- 텍스처 해상도와 라이선스 상태 확인.
- 카탈로그 제품이면 SKU/가격/재고와 연결.

## 5. Iterative Regeneration

사용자의 수정 예:
- “이 의자는 더 얇게.”
- “이쪽으로 30cm만 옮겨.”
- “금속 말고 밝은 원목.”
- “예산 500만 원 안에서 맞춰.”

시스템 처리:
1. 기존 layout_variant를 복제.
2. 변경 제약만 업데이트.
3. 영향 범위가 작은 경우 부분 재생성.
4. 전체 공간 검증을 다시 실행.
5. 이전 버전과 차이점 표시.

## 6. 모델 제공자 연동 정책

- 외부 3D 생성 API는 다중 벤더 구조로 추상화한다.
- 특정 벤더 장애 또는 품질 저하에 대비한다.
- 입력/출력 포맷은 내부 표준으로 변환한다.
- 고객 민감 공간 데이터는 가능하면 모델 제공자에 직접 전달하지 않고, 추상화된 제약만 전달한다.
