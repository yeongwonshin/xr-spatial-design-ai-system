# XR Interaction Design

## 1. 입력 모드

### A. Pen Mode
- 바닥/벽/공중에 3D 선 그리기.
- 버튼으로 선/영역/오브젝트/동선 모드 전환.
- 압력 또는 버튼 입력으로 선 두께와 중요도 표현.

### B. Hand Mode
- 생성된 오브젝트 잡기, 이동, 회전, 스케일.
- 두 손으로 그룹 선택.
- 손바닥 제스처로 빠른 삭제/되돌리기.

### C. Voice Mode
- “이 선을 바 테이블로 바꿔줘.”
- “여기 통로는 1미터 이상 남겨.”
- “더 고급스럽게, 조명은 따뜻하게.”
- “예산 800만 원 안에서 다시 배치해.”

### D. Text/Panel Mode
- 정확한 치수 입력.
- SKU/브랜드/예산 선택.
- 스타일 프리셋 선택.

## 2. 핵심 UX 패턴

### Sketch-to-Object
1. 사용자가 대략적 부피를 그림.
2. 시스템이 “의자/소파/테이블/선반?” 후보를 표시.
3. 사용자가 음성으로 확정.
4. AI가 3D 오브젝트 생성/추천.

### Zone Painting
1. 바닥에 영역을 칠함.
2. “여기는 카페 좌석존.”
3. 시스템이 여러 좌석 배치를 제안.

### Constraint Pin
1. 특정 위치에 핀을 꽂음.
2. “여기는 비워둬.”
3. 해당 영역이 hard exclusion zone으로 저장.

### Conversational Revision
- “방금 것보다 더 낮게.”
- “왼쪽 벽에 붙여.”
- “세 번째 안을 유지하고 조명만 바꿔.”

### Before/After Layer
- 원래 공간, 생성안, 수정안, 확정안을 레이어로 전환.

## 3. 화면 구성

### XR 메인 뷰
- 실제 공간 메쉬.
- 생성된 3D 오브젝트.
- 검증 경고 오버레이.
- 사용자별 컬러 스케치.
- 미니맵/평면도 뷰.

### Floating Control Panel
- 스타일 프리셋.
- 생성 후보.
- 예산 슬라이더.
- 검증 결과.
- 견적 요약.

### Review Mode
- 버전 비교.
- 코멘트 핀.
- 승인/반려 버튼.
- 견적 PDF 내보내기.

## 4. 전문 UX 요구사항

- 디자이너가 AI 결과를 “잠금” 처리할 수 있어야 한다.
- 고객이 실수로 배치를 변경하지 못하도록 권한 분리.
- 모든 생성/수정/승인 이력을 저장.
- XR 기기가 없는 고객도 Web Viewer에서 확인 가능.
- 회의 중 네트워크 지연 시 로컬 캐시 프리뷰 제공.
