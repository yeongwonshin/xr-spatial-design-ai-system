# Security & Privacy

## 1. 보호 대상 데이터

- 고객 주거/상업 공간 스캔 데이터.
- 위치 정보와 평면 구조.
- 고객명, 연락처, 견적 금액.
- 브랜드 카탈로그 가격/재고.
- 프로젝트 계약/승인 로그.
- 생성 프롬프트와 디자인 히스토리.

## 2. 보안 요구사항

| 영역 | 요구사항 |
|---|---|
| 인증 | SSO/SAML/OAuth, MFA 옵션 |
| 권한 | 조직/프로젝트/역할 기반 RBAC |
| 암호화 | 전송 TLS, 저장 AES-256 수준 암호화 |
| 감사 | 모든 공유/승인/export 로그 저장 |
| 데이터 리전 | 엔터프라이즈 고객별 리전 선택 |
| 외부 API | 최소 정보 전송, 벤더별 DPA 필요 |
| 공유 링크 | 만료일, 비밀번호, 워터마크, 다운로드 제한 |

## 3. 프라이버시 처리

- 얼굴, 가족사진, 문서, 화면, 주소 표시 자동 감지/블러.
- 외부 3D 생성 API에는 원본 공간 스캔 대신 추상화된 제약조건 우선 전달.
- 고객이 프로젝트 종료 후 데이터 삭제를 요청할 수 있어야 함.
- 학습 데이터 사용 여부를 조직별로 명시적으로 opt-in 처리.

## 4. 엔터프라이즈 옵션

- 전용 VPC 배포.
- 사내 에셋 라이브러리 비공개 운영.
- 감사 로그 SIEM 연동.
- 고객 전용 모델/룰셋.
- 온프레미스 또는 프라이빗 클라우드 추론.

## 5. 위험 시나리오와 대응

| 위험 | 대응 |
|---|---|
| 고객 집 구조 유출 | 암호화, 접근 로그, 공유 링크 제한 |
| 브랜드 가격정보 노출 | 카탈로그 권한 분리, 워터마크 |
| 외부 모델 제공자 데이터 재사용 | 계약/DPA, 데이터 최소화, 익명화 |
| 생성 결과의 저작권 분쟁 | 라이선스 태그, 금지 프롬프트, 에셋 출처 추적 |
| 잘못된 시공 판단 | 전문가 검토 라벨, 법규/구조 검토 면책 표시 |
