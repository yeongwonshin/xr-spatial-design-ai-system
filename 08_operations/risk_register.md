# Risk Register

| 위험 | 가능성 | 영향 | 대응 |
|---|---:|---:|---|
| 3D 생성 품질이 B2B 기준 미달 | 높음 | 높음 | 카탈로그/라이브러리 우선, 생성은 보조로 사용 |
| 생성 속도가 현장 미팅에 느림 | 높음 | 중간 | 프리뷰-고품질 2단계, 캐싱, 비동기 처리 |
| 공간 스캔 치수 오차 | 중간 | 높음 | 기준 길이 보정, 수동 수정, 신뢰도 라벨 |
| 외부 API 비용 증가 | 중간 | 높음 | 벤더 추상화, 사용량 제한, 캐시, 자체 모델 장기화 |
| CAD/BIM 연동 난이도 | 높음 | 중간 | 초기에는 GLB/USDZ/PDF, 이후 IFC/DWG 단계적 지원 |
| 고객 데이터 프라이버시 우려 | 중간 | 높음 | 암호화, 익명화, 데이터 리전, 외부 전송 최소화 |
| 법규/시공 책임 문제 | 중간 | 높음 | 전문가 검토 라벨, 면책, 규칙 엔진, 승인 워크플로우 |
| XR 하드웨어 보급 제한 | 중간 | 중간 | iPad/Web viewer 병행, 현장 영업용 장비 렌탈 |
| 디자이너의 AI 거부감 | 중간 | 중간 | AI를 대체자가 아닌 보조자로 포지셔닝, 수동 통제권 제공 |
| 브랜드 카탈로그 확보 어려움 | 중간 | 높음 | 초기 일반 에셋+파일럿 브랜드, 3D 변환 패키지 제공 |

## 최우선 리스크

1. 생성 결과의 실제 사용성.
2. 공간 치수 정확도.
3. 현장 미팅 속도.
4. B2B 견적 신뢰도.
5. 고객 데이터 보안.

## 리스크 완화 제품 전략

- “AI가 모든 것을 만든다”가 아니라 “AI가 후보를 만들고 전문 검증이 붙는다”로 설계한다.
- 생성형 에셋보다 실제 구매 가능한 카탈로그 에셋을 우선 노출한다.
- 고객에게 보이는 출력물에는 항상 신뢰도와 검증 상태를 표시한다.
